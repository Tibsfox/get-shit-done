Create a comprehensive logging reference guide following GSD's hybrid documentation pattern (prose + code examples, XML-tagged sections).

Structure (follow model-profiles.md style):

1. **Quick Start** (~30 lines)
   - Default behavior (INFO level, syslog output)
   - Enable DEBUG: `GSD_LOG_LEVEL=4` or config
   - View logs: `journalctl --user -t gsd -f`

2. **Log Levels** (~40 lines)
   - Table with all 6 levels (OFF=0 through TRACE=5)
   - Use case for each level
   - When to use each (ERROR: critical failures, WARN: recoverable issues, INFO: workflow milestones, DEBUG: detailed operations, TRACE: complete transparency)

3. **Syslog Primer** (~30 lines)
   - Brief explanation of syslog and journalctl for users unfamiliar with Unix logging
   - GSD uses LOCAL0 facility by default
   - Viewing commands with expected output examples

4. **Configuration** (~50 lines)
   - Configuration precedence: env > project > global > defaults
   - Environment variables: GSD_LOG_LEVEL, GSD_LOG_SYSLOG, GSD_LOG_FACILITY
   - Project config (.planning/config.json) example
   - Global config (~/.claude/gsd-config.json) example
   - Copy-pasteable JSON snippets

5. **Viewing Logs** (~40 lines)
   - journalctl patterns for common use cases:
     - Real-time streaming: `journalctl --user -t gsd -f`
     - Last N entries: `journalctl --user -t gsd -n 50`
     - Filter by priority: `journalctl --user -t gsd -p info`
     - Filter by phase: `journalctl --user -t gsd | grep "phase=05"`
     - JSON output: `journalctl --user -t gsd -o json | jq '.MESSAGE'`

6. **Troubleshooting** (~60 lines)
   - Symptom-Diagnosis-Solution format for common issues:
     - "No logs appearing" - check syslog enabled, level not OFF
     - "Too many logs" - lower level from DEBUG to INFO
     - "Can't find phase logs" - grep patterns for phase/wave
     - "journalctl permission denied" - use --user flag
   - Include diagnostic commands with expected output

7. **Cross-References** (~10 lines)
   - Link to verification-logging.md for verification patterns
   - Link to agent files for logging specifications
   - Link to workflow files for orchestrator logging
