  <files>agents/gsd-verifier.md</files>
  <action>
Add a new subsection "### Audit Trail Considerations" at the end of the `<logging>` section that documents:

1. **Correlation requirements:**
   - All verification events include `agent_id` for session correlation
   - Re-verification links to previous via `previous_verification_id` field
   - Gap closure plans reference originating `verification_id` in their logs

2. **Immutability guidance:**
   - Log entries are append-only (syslog provides this)
   - Never modify previous verification logs
   - Each re-verification creates new entries, doesn't update old ones

3. **Chain of custody:**
   - Verification start logs capture must_haves snapshot
   - Gap detection logs capture full gap context
   - Verification outcome logs provide closure

4. **Querying patterns (prose guidance):**
   - Filter by agent_id to see single verification session
   - Filter by phase to see all verifications of a phase
   - Filter by status=gaps_found to find incomplete verifications
   - Sort by timestamp to see progression across iterations
