Update hooks/gsd-check-update.js to log update check operations at DEBUG level.

1. Add logger import with graceful fallback near the top (after existing requires):
   ```javascript
   // Import logger - may not be initialized if this runs before gsd-log-init
   let logger = null;
   try {
     const { getLogger } = require('../lib');
     logger = getLogger();
   } catch (e) {
     // Logger not available - continue without logging
   }
   ```

2. Add helper function for safe logging:
   ```javascript
   function logDebug(message, context = {}) {
     if (logger && !logger.isOff) {
       logger.debug(message, { ...context, hook: 'check-update' });
     }
   }
   ```

3. Add logging BEFORE spawning background process (after cache dir creation):
   ```javascript
   // Log update check initiation at DEBUG level
   logDebug('Update check initiated', {
     cacheFile,
     projectVersionFile,
     globalVersionFile
   });
   ```

4. Add logging AFTER spawning background process (before child.unref()):
   ```javascript
   // Log that background check was spawned
   logDebug('Update check spawned in background');
   ```

5. Key behaviors per research:
   - Import uses try/catch to handle case where logger not yet initialized
   - Use DEBUG level (4) - not visible by default, only when debugging
   - Include 'hook' field in context for filtering
   - Do NOT log from the spawned child process (it doesn't share state)
   - Logging is fire-and-forget, never blocks the hook
